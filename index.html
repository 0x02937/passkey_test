Passkey Tester

Standard (ES256) and Ed25519 (-8) passkey creation & authentication across environments

⚠ WebAuthn requires a secure context. Serve this file via a local server: python -m http.server 8080 then open http://localhost:8080
⚠ WebAuthn / PublicKeyCredential is not supported in this browser.
Standard Passkey Sanity check — ES256 (-7) + RS256 (-257), works on all platforms
Step 1

Create Standard Passkey

Registers a discoverable passkey using the default algorithms supported everywhere, including Windows Hello. Use this to confirm WebAuthn is working before testing -8.

pubKeyCredParams: [{ alg: -7 }, { alg: -257 }] // ES256, RS256
Step 2

Check Standard Passkey

Opens the browser passkey picker. Use QR code to authenticate from another device.

Ed25519 Passkey
Step 1

Create Passkey (-8)

Registers a discoverable passkey specifying only alg -8 (EdDSA / Ed25519). Will fail on Windows Hello — use an iPhone, Android, or macOS device.

pubKeyCredParams: [{ type: "public-key", alg: -8 }]
Step 2

Check / Authenticate

Opens the browser passkey picker. Choose QR code to scan with an iPhone or Android device. The full decoded response is shown below so you can verify what your passkey actually returned.
